Finalidade e âmbito

O objetivo desta política é garantir que a organização estabeleça objetivos, planos e procedimentos de modo que Uma grande interrupção nas principais atividades comerciais da organização é minimizada.

Esta política se aplica a toda infraestrutura e dados no programa de segurança de informações da organização.

Esta política se aplica a toda a gerência, funcionários e fornecedores envolvidos em decisões e processos afetando a continuidade dos negócios da organização.Esta política deve ser disponibilizada prontamente para todos os quais aplica-se a.

Fundo

  • O sucesso da organização depende da preservação de operações comerciais críticas e essenciais funções usadas para fornecer produtos e serviços essenciais. O objetivo desta política é definir os critérios para continuidade das operações comerciais da organização em caso de interrupção. Especificamente, este documento define:
    • A estrutura e a autoridade para garantir a resiliência dos negócios dos principais processos e sistemas.
    • Os requisitos para os esforços para gerenciar através de um desastre ou outro evento disruptivo quando a necessidade surge.
    • Os critérios para retomar de maneira eficiente e efetiva operações comerciais normais após uma interrupção.
  • Neste documento, aplicam-se as seguintes definições:
    • Análise/Avaliação de Impacto dos Negócios - Um exercício que determina o impacto de perder o apoio de Qualquer recurso para uma empresa estabelece a escalada dessa perda ao longo do tempo, identifica o mínimo Recursos necessários para retornar a um nível normal de operação e prioriza a recuperação de processos e o sistema de suporte.
    • Plano de recuperação de desastres - um conjunto de recursos humanos, físicos, técnicos e processuais para retornar a um nível normal de operação, dentro de um tempo e custo definidos, quando uma atividade é interrompida por um emergência ou desastre.
    • Objetivo do tempo de recuperação - a quantidade de tempo permitida para a recuperação de uma função ou recurso comercial a um nível normal após ocorrer um desastre ou interrupção.
    • Objetivo do ponto de recuperação - determinado com base na perda de dados aceitável no caso de interrupção de operações.

Política

  • Avaliação de riscos empresariais e análise de impacto empresarial
    • Cada gerente é obrigado a realizar uma avaliação de risco comercial e análise de impacto nos negócios para cada chave sistema de negócios em sua área de responsabilidade.
    • A avaliação de risco comercial deve identificar e definir a criticidade dos principais sistemas de negócios e o Repositórios que contêm os dados relevantes e necessários para o principal sistema de negócios.
    • A avaliação de riscos empresariais deve definir e documentar o Plano de Recuperação de Desastres (DRP) para sua área de atuação. responsabilidade.  Cada DRP deverá incluir:
      • Principais processos de negócios.
      • Risco aplicável à disponibilidade.
      • Priorização da recuperação.
      • Objetivos do tempo de recuperação (RTOs).
      • Objetivos do ponto de recuperação (RPOS).
  • Plano de Recuperação de Desastres
    • Cada sistema de negócios importante deve ter um DRP documentado para fornecer orientação quando hardware, software ou As redes tornam -se criticamente disfuncionais ou deixam de funcionar (interrupções a curto e longo prazo).
    • Cada DRP deve incluir uma explicação da magnitude da informação ou sistema de indisponibilidade no evento de uma interrupção e o processo que seria implementado para continuar as operações de negócios durante o interrupção.Onde viável, o DRP deve considerar o uso de operações alternativas de computador fora do local (Sites frios, quentes e quentes).
    • Cada plano deve ser revisado contra a estratégia, objetivos, cultura e ética da organização, bem como requisitos políticos, legais, estatutários e regulatórios.
    • Each DRP must include:
      • Um plano de operações de modo de emergência para operações continuadas em caso de hardware temporário, software ou interrupções de rede.
      • Um plano de recuperação para devolver funções e serviços de negócios para operações normais no local.
      • Procedimentos para testes, revisão e revisões periódicas do DRP para todos os negócios afetados sistemas, como grupo e/ou individualmente.
  • Planos de backup e Restauração de Dados
    • Cada proprietário do sistema deve implementar um plano de backup e restauração de dados.
    • Cada plano de backup e restauração de dados deve identificar:
      • O custodiante de dados para o sistema.
      • O cronograma de backup de cada sistema.
      • Onde a mídia de backup deve ser armazenada e protegida, bem como como o acesso é mantido.
      • Que podem remover a mídia de backup e transferi -lo para o armazenamento.
      • Procedimentos de restauração apropriados para restaurar os principais dados do sistema de negócios da mídia de backup para o sistema.
      • O plano de teste de restauração e a frequência dos testes para confirmar a eficácia do plano.
      • O método para restaurar a mídia de backup criptografada.
